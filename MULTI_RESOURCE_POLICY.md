# Multi-Resource Policy

이 문서는 Numbers 자동 생성 파이프라인에서 **과목별 기본 리소스 조합 정책**을 정의한다.

핵심 목적은 다음과 같다.
- 매번 사용자 지시로 `이 과목은 보조 교재를 같이 쓴다`를 반복하지 않게 한다.
- 새로운 채팅에서도 동일한 기본 입력 구성을 재현할 수 있게 한다.
- 모델 추론이 아니라 운영 규칙으로 멀티리소스 여부를 먼저 확정한다.

이 문서는 “추가로 붙일 수 있다” 수준의 권고가 아니라, **기본 운영 규칙**이다.

## 1. 기본 원칙

1. 멀티리소스 여부는 우선적으로 **과목 정책**으로 결정한다.
2. 단원 config에 `resources`가 명시되어 있으면 그것이 최우선이다.
3. 단원 config에 `resources`가 없더라도, 과목 정책상 멀티리소스 과목이면 기본 리소스 조합을 자동 주입하는 방향을 권장한다.
4. AI 모델은 “어떤 리소스를 기본 포함할지”를 결정하는 주체가 아니다.
5. AI 모델은 이미 확정된 리소스를 바탕으로 분석과 활동 생성을 수행한다.

## 2. 우선순위

리소스 결정 우선순위는 아래와 같다.

1. 단원 config의 명시적 `resources`
2. 과목별 기본 멀티리소스 정책
3. 단일 교과서 fallback

즉, 명시적 config가 없고 정책도 없을 때만 단일 교과서로 간다.

## 3. 기본 운영 정책

### 3.1 사회
- 기본 구성:
  - 사회 교과서
  - 사회과부도
- 해석:
  - 사회는 기본적으로 멀티리소스 과목으로 본다.
  - 사회과부도는 기본 포함 대상이다.
  - 단, 특정 단원에서 실제로 매칭되는 부도 자료가 없으면 최종 runtime config에서 빠질 수 있다.

### 3.2 수학
- 기본 구성:
  - 수학 교과서
  - 수학익힘
- 해석:
  - 수학은 기본적으로 멀티리소스 과목으로 본다.
  - 수학익힘은 기본 포함 대상이다.

### 3.3 과학
- 기본 구성:
  - 과학 교과서
  - 실험관찰
- 해석:
  - 과학은 기본적으로 멀티리소스 과목으로 본다.
  - 실험관찰은 기본 포함 대상이다.

### 3.4 그 외 과목
- 기본 구성:
  - 주 교과서 단일 리소스
- 해석:
  - 별도 정책이 추가되기 전까지는 단일 리소스로 본다.

## 4. 리소스 역할 규칙

### 주 교과서
- `kind: "textbook"`
- 활동 생성과 lesson analysis의 주 근거가 된다.

### 보조 교재
- `kind: "supplement"`
- 기본적으로 Numbers 배치용 자료다.
- 필요 시에만 Gemini 입력 증거로 확장한다.
- 보조 교재가 있다고 해서 활동 생성이 반드시 그 자료를 직접 참조해야 하는 것은 아니다.

## 5. Config 작성 규칙

### 5.1 명시적 멀티리소스 config가 있는 경우
- `resources`와 `sources`를 config에 직접 적는다.
- 이 경우 정책 문서는 참고만 하고, config가 우선한다.

### 5.2 단일 리소스 config만 있는 경우
- 과목 정책이 멀티리소스이면, 장기적으로는 실행기에서 기본 리소스를 자동 주입하도록 개선하는 것이 맞다.
- 그 전까지는 멀티리소스용 sample/demo config를 별도로 관리한다.

## 6. 매칭 실패 해석 규칙

보조 교재가 최종 Numbers에 안 들어갔다고 해서 항상 정책 위반은 아니다.

아래 순서로 판정한다.

1. 애초에 config가 단일 리소스였는가
2. runtime config에 보조 리소스 `sources`가 들어갔는가
3. 실제 `source_ranges`로 확정되었는가
4. 최종 `render_manifest.json`에 `supplement_card`가 생성되었는가

즉, 누락 원인은 아래 셋 중 하나다.
- 단일 리소스 config로 실행함
- 보조 교재 query가 매칭되지 않음
- review 단계에서 보조 자료가 제외됨

## 7. 사용자 지시와의 관계

기본적으로 사용자가 매번 아래처럼 지시할 필요는 없다.
- `이 과목은 교과서가 여러 개야`
- `사회과부도도 같이 넣어줘`

이런 지시는 예외 상황에서만 필요하다.

예외:
- 특정 단원만 보조 교재를 제외하고 싶을 때
- 기본 보조 교재 외에 추가 자료를 붙일 때
- 매칭 실패를 수동 override 할 때

## 8. 모델 자동 판단의 위치

모델이 자동으로 멀티리소스 여부를 판단하게 하는 것은 보조 기능으로만 허용한다.

허용되는 역할:
- 사용 가능한 PDF 목록을 보고 `사회과부도`, `수학익힘`, `실험관찰` 후보를 제안
- 누락된 멀티리소스 config를 추천

허용하지 않는 역할:
- 운영 정책을 대신 결정
- 명시적 config보다 우선하여 리소스를 바꾸는 것

즉, 모델은 **제안자**일 수는 있지만 **정책 결정자**가 되면 안 된다.

## 9. 권장 다음 단계

이 문서를 실제 운영 규칙으로 만들기 위한 다음 단계는 아래와 같다.

1. 과목별 기본 리소스 매핑 파일 도입
2. 단일 리소스 config 실행 시 과목 정책을 참조하는 fallback 추가
3. runtime config에 “정책에 의해 자동 주입된 리소스” 여부 기록
4. review report에 supplement 매칭 실패 원인 명시

## 10. 현재 결론

현재 운영 기준에서:
- 사회는 기본적으로 `사회 + 사회과부도`
- 수학은 기본적으로 `수학 + 수학익힘`
- 과학은 기본적으로 `과학 + 실험관찰`

으로 보는 것이 맞다.

따라서 새 채팅이나 새 작업에서도, 이 과목들은 원칙적으로 멀티리소스 과목으로 취급하는 것이 맞다.
