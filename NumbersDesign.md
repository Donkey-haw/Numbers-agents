# Numbers Design

이 문서는 Gemini CLI와 로컬 렌더러가 Numbers용 활동 카드를 만들 때 따라야 하는 **상위 디자인 규칙**을 정의한다.

핵심 원칙은 단순하다.
- 이 문서는 템플릿 모음집이 아니다.
- 이 문서는 **자유 생성 HTML이 반드시 지켜야 하는 제약 문서**다.
- 개별 템플릿 예시는 [templateplan.md](/Users/jonyeock/Desktop/Tool/NumbersAuto/templateplan.md) 와 `templates/`가 맡는다.

즉, Gemini는 이 문서의 규칙만 지키면 새로운 레이아웃을 설계할 수 있다. 기존 템플릿과 똑같이 만들 필요는 없다.

---

## 1. 문서의 역할

이 문서가 결정하는 것:
- 학생용 Numbers 카드가 반드시 지켜야 하는 시각적 제약
- 객체별 역할과 수업 흐름
- 학생 필기 공간 규칙
- 자유 생성 HTML의 허용 범위와 금지 사항

이 문서가 결정하지 않는 것:
- 특정 활동이 반드시 어떤 템플릿 이름을 써야 하는지
- 특정 활동이 반드시 3컬럼, 4사분면, 스펙트럼 구조여야 하는지
- 특정 배지 문구나 특정 HEX 컬러를 항상 써야 하는지

## 2. 최상위 목표

[ACTIVITY_RULE.md](/Users/jonyeock/Desktop/Tool/NumbersAuto/ACTIVITY_RULE.md) 기준으로 Numbers 수업 자료는 아래 목표를 지원해야 한다.

1. 학생의 학습 과정이 누적되고 기록될 것
2. Apple Pencil 기반 필기가 편할 것
3. 무한한 가로 캔버스를 활용해 수업 전, 수업 중, 수업 후 흐름이 드러날 것
4. 창작형, 사고형, 기초 기능형 활동을 모두 수용할 수 있을 것
5. 교과서, 활동, 참고자료, 학습지, AI 연계가 하나의 수업 흐름 안에 배치될 것

디자인은 예뻐 보이는 것보다 이 목표를 더 우선한다.

## 3. 절대 제약

아래 규칙은 자유 생성 HTML이 반드시 지켜야 한다.

### 3.1 캔버스와 배경
- `body` 기본 폭은 `1600px`를 사용한다.
- 카드 바깥 배경은 `transparent` 또는 `white`만 사용한다.
- Numbers 기본 흰 배경과 충돌하는 회색 캔버스는 금지한다.
- 너무 긴 가로 배너형 카드보다, 세로 필기 공간이 충분한 카드가 우선이다.

### 3.2 작성 영역
- 학생이 실제로 쓰는 영역은 반드시 **흰색 배경**이어야 한다.
- 작성 영역은 반드시 `2px solid #333` 수준의 강한 테두리를 사용한다.
- 얇은 회색 테두리, 점선 테두리, 흐린 그림자만 있는 입력 박스는 금지한다.
- 작성 영역이 단순 장식 요소보다 시각적으로 더 강해야 한다.

### 3.3 높이와 공간 예산
- 일반 작성 영역 최소 높이: `220px`
- 주요 작성 영역 최소 높이: `420px`
- 서술 중심 메인 영역 권장 높이: `520px+`
- 자유 생성 카드 전체 높이는 보통 `1200px~1800px` 범위를 권장한다.
- 설명문보다 학생 입력 공간이 더 큰 비중을 가져야 한다.

### 3.4 가독성
- 작은 폰트, 과밀한 문단, 촘촘한 4단 이상 그리드는 금지한다.
- 제목, 안내문, 작성 영역 구분이 한눈에 보여야 한다.
- 색은 식별용으로만 쓰고, 읽기 자체는 텍스트와 여백으로 해결한다.

## 4. 객체 중심 설계

[ACTIVITY_RULE.md](/Users/jonyeock/Desktop/Tool/NumbersAuto/ACTIVITY_RULE.md) 기준으로 한 차시 시트는 아래 객체들로 구성된다.

### 4.1 수업 전 객체
- 교과서
- 추가자료

규칙:
- 시트의 가장 왼쪽에서 시작한다.
- 교과서 객체는 항상 기준점 역할을 한다.
- 추가자료가 있으면 교과서 뒤 또는 교과서 하단 확장 영역에 둔다.

### 4.2 수업 중 객체
- 학습 노트
- 활동 영역

규칙:
- 시트의 중간 영역에 둔다.
- 학생이 가장 오래 머무는 구간이므로 필기 공간을 가장 크게 준다.
- 학습 노트와 활동은 서로 역할이 달라야 한다.

### 4.3 수업 후 객체
- 참고 자료
- 학습지
- AI 코스웨어 연계

규칙:
- 시트의 오른쪽 영역에 둔다.
- 참고 자료는 “모든 것을 읽게 하는 영역”이 아니라 “필요할 때 접근하는 영역”이어야 한다.
- 학습지는 정리, 적용, 평가 역할이 분명해야 한다.
- AI 코스웨어 연계는 링크와 결과 기록 공간이 함께 있어야 한다.

## 5. 시트 배치 원칙

### 5.1 기본 흐름
- 수업 전 -> 수업 중 -> 수업 후 정리 순으로 **가로 배치**한다.
- 객체 간 기본 간격은 `200pt`를 기준으로 잡는다.
- 교과서만 덩그러니 두지 말고, 같은 시트 안에서 수업 흐름이 읽히게 배치한다.

### 5.2 객체 크기
- 기본 객체 크기 기준은 `가로 1000pt`, `세로 700pt`다.
- 단, 실제로는 내용에 따라 유기적으로 조절한다.
- 서술형, 창작형, 사고형 활동은 세로를 더 크게 준다.
- 자료 제공용 객체는 상대적으로 작아도 되지만, 학생이 써야 하는 객체는 작아지면 안 된다.

### 5.3 레이아웃 해석 규칙
- “가로 배치”는 모든 카드가 똑같은 폭이어야 한다는 뜻이 아니다.
- 카드마다 역할이 다르면 폭과 높이가 달라도 된다.
- 대신 전체 시트에서 흐름과 위계는 유지되어야 한다.

## 6. 객체별 필수 요소

### 6.1 교과서
- 교과서 페이지 자체가 주인공이다.
- 장식보다 페이지 판독성이 우선이다.
- 교과서 카드가 활동 카드보다 지나치게 약해 보이면 안 된다.

### 6.2 학습 노트
- 기본 성격은 **줄 없는 백지 노트**다.
- 용도는 학생 자유 필기와 교사 설명 정리다.
- 상단에는 차시와 학습 목표가 보여야 한다.
- 하단 또는 측면에는 메모, 질문, 핵심 정리 공간을 둘 수 있다.
- 노트는 “문제풀이 양식”처럼 보이면 안 된다.

### 6.3 활동 영역
- 교과서 활동의 보완, 심화, 확장 역할을 한다.
- 활동은 아래 3축 중 하나 이상에 속해야 한다.
  - 창작형
  - 사고 확장형
  - 기초 기능 연습형

#### 창작형 활동
- 그림, 글, 영상, 설계, 표현 중심
- 충분한 자유 공간 필요
- 가이드라인과 예시가 있어야 한다

#### 사고 확장형 활동
- 근거, 비교, 연결, 판단, 설명이 드러나야 한다
- 사고 과정이 학생 필기로 남도록 구조화해야 한다
- 단순 답만 적는 박스 나열은 부족하다

#### 기초 기능 연습형 활동
- 반복 풀이, 용어 정리, 조사, 분류 등
- 충분한 연습 칸이 필요하다
- 작은 박스를 여러 개 둘 수 있지만 과밀하면 안 된다

### 6.4 참고 자료
- 글, 그림, 사진, 유튜브, 웹사이트, 사료 등
- 학생이 필요할 때 바로 접근 가능해야 한다
- 참고 자료 자체가 메인 필기 공간을 압도하면 안 된다

### 6.5 학습지
- 활동 결과를 바탕으로 적용하거나 정리하는 공간이다
- 답을 쓰는 칸보다 사고를 드러내는 칸이 더 중요할 수 있다
- 필요하면 답안지/예시 위치를 별도 구획으로 분리한다

### 6.6 AI 코스웨어 연계
- 버튼만 두면 안 된다
- 결과 기록 공간이 함께 있어야 한다
- 학생이 AI 결과를 보고 다시 자기 생각을 정리할 수 있어야 한다

## 7. 자유 생성 HTML 규칙

Gemini가 HTML을 직접 생성할 때는 아래 순서로 판단한다.

1. 이번 차시에 어떤 객체가 필요한가
2. 각 객체가 수업 전, 수업 중, 수업 후 중 어디에 속하는가
3. 활동이 창작형, 사고형, 기초형 중 무엇에 가까운가
4. 학생이 실제로 무엇을 쓰거나 그릴 것인가
5. 그 목적에 맞는 레이아웃을 새로 설계할 것인가, 기존 패밀리를 응용할 것인가

### 허용
- 새로운 카드 구조 설계
- 비대칭 레이아웃
- 2단, 3단, 분할, 지도형, 캔버스형 구성
- 하나의 카드 안에서 자료와 필기 영역을 새롭게 조합

### 금지
- 설명문만 많고 쓸 곳이 거의 없는 카드
- 지나치게 템플릿스러운 얇은 폼 UI
- 필기 공간보다 장식이 큰 카드
- 배지, 제목, 안내문, 필기영역의 위계가 없는 카드
- 회색 바탕 전체 캔버스

## 8. 배지와 색상 시스템

배지와 색은 유지할 수 있지만, **고정 정답 시스템은 아니다**.

### 원칙
- 배지는 객체 역할이나 활동 성격을 빠르게 알려 주기 위한 장치다.
- 배지 텍스트와 색은 카드 목적에 맞게 바뀔 수 있다.
- 하나의 카드에 배지가 없어도 역할 구분이 충분하면 허용된다.
- 다만 카드 역할이 불분명해지면 안 된다.

### 유지해야 하는 성격
- 명확한 식별
- 높은 대비
- 과한 장식보다 빠른 해석

### 권장
- 둥근 pill 형태
- 볼드 텍스트
- 밝은 배경 위에 시각적 포인트 색상

### 비권장
- 모든 카드가 같은 배지 구조를 그대로 복제하는 것
- 색상 체계가 역할보다 단순한 장식으로 쓰이는 것

## 9. 레이아웃 패밀리

아래는 **정답 템플릿**이 아니라 **레이아웃 패밀리 예시**다.

### 9.1 노트형
- 큰 자유 필기 공간
- 보조 메모 또는 질문 영역
- 수업 중 설명 정리용

### 9.2 탐구형
- 자료 제시 + 질문 + 근거 정리
- 사고 과정 드러내기

### 9.3 창작형
- 넓은 표현 공간
- 가이드와 예시 포함 가능

### 9.4 비교형
- 두세 관점, 사례, 개념을 대비
- 구분 라벨은 필요하지만 구조는 자유

### 9.5 자료 활용형
- 사진, 영상, 링크, 사료와 함께 쓰는 구조
- 참고 자료가 필기 공간을 압도하지 않아야 함

### 9.6 평가/정리형
- 학습지, 확인 문제, 정리 카드
- 짧은 답과 긴 서술이 혼합 가능

Gemini는 이 패밀리를 참고할 수 있지만, 이 이름을 그대로 카드 유형으로 복제할 필요는 없다.

## 10. 템플릿 문서와의 관계

[templateplan.md](/Users/jonyeock/Desktop/Tool/NumbersAuto/templateplan.md)는 기존 템플릿을 개선하는 참고 문서다.

따라서:
- `templateplan.md`는 **예시와 발전 계획**
- 이 문서는 **자유 생성용 상위 규칙**

새로운 카드가 `templateplan.md`의 기존 예시와 다르더라도, 이 문서의 제약과 [ACTIVITY_RULE.md](/Users/jonyeock/Desktop/Tool/NumbersAuto/ACTIVITY_RULE.md)를 만족하면 허용된다.

## 11. 최종 체크리스트

생성된 카드가 아래를 만족하면 통과다.

- 수업 흐름상 어디에 놓이는지 보이는가
- 객체 역할이 분명한가
- 학생이 쓸 공간이 충분한가
- 필기 영역이 흰색 + 강한 검정 테두리인가
- 장식보다 학습 행위가 중심인가
- 창작형/사고형/기초형 활동의 의도가 드러나는가
- Numbers 가로 캔버스에서 자연스럽게 이어지는가
- 기존 템플릿을 복제하지 않아도 이해 가능한가
